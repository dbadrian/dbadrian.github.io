eknog - A framework for embedding knowledge graphs

Find eknog on Github! [soon...]

eknog (Embedding Knowledge Graphs) is a Python+Tensorflow project allowing for the easy training of latent feature representations (=embeddings) of knowledge graphs (KG). It provides reference implementations of some of the recent and popular models (e.g., TransE, TorusE, DistMult, ComplEx) and it should be reasonably easy to implement models which are simple variations of these. There is only very limited support for CNN-models, like ConvE, which require special training procedures (e.g., what they call 1-N evaluation).

It is currently in a review and refactoring process as preparation for a release together with a publication - please bear with me, if it is temporarily unavailable.

This was developed at Mercateo - thanks!

Muninn - A tool for maintaining dotfiles with extras :)

Find Muninn on Github!

Muninn is a small personal project to maintain, backup, synchronize dotfiles, maintain tools (e.g., sublime, zsh), as well as install various programs and libraries (e.g., compile Tensorflow from source with appropriate optimizations enabled). It never was and is not meant to be a general, ready plug'n'play tool for everyone, but strong inclination towards Arch Linux support (and using Anaconda), as that is my primary setup. Shouldn't be too hard our just copy-paste the dotfiles (which are kept in an extra repository).

Various tasks, which include symlinking dotfiles or compiling a program, are defined as packages (realized a python module). The syntax is still changing, but generally it is quite heavily inspired by how PKGBUILD files are defined for Arch Linux. One can define dependencies between different Muninn-packages, as well as Arch Linux packages (hence the strong bias towards Arch). For example, an empty package, but with many Arch Linux dependencies can be used to quickly install common packages for Arch Linux after a fresh installation.

However, I am rewriting most of the code from scratch right now, which will soften those requirements.

EBOF - (Bio-inspired) Event-based Optical Flow

Find EBOF on Github!

As part of a university course, Adam Kosiorek (check out his sweet blog on VAE!), Johannes Rausch, and myself implemented a bio-inspired optical flow for event-based cameras/dynamic vision sensors.

The original publication by Brosch et al., proposes to use spatio-temporal Gabor filters to calculate the optical flow. The goal was to provide an open-source reference implementation, that focuses on efficiency in order to evaluate the real-time capability of this algorithm. For a write-up of the implementation, see our report.

Since I am big fan of Paradise Lost, the John Milton Reading Room became a great resource of understanding the epic better. Generally, I prefer reading a printed version of things, however, this is not an available option. Thus I made this little python-based crawler, generating a tex version.
